Este documento describe la arquitectura, archivos, flujo de datos, integraciones externas y un tutorial completo para ejecutar y operar la aplicación Scoreboard.

Electron OBS WebSocket Challonge start.gg Twitch / Nightbot Stream Deck

1. ¿Qué es este proyecto?

Es una app de escritorio centrada en la producción para torneos y streams de fighting games. Permite editar un scoreboard en tiempo real, sincronizar datos desde brackets en vivo, controlar escenas de OBS, publicar mensajes automatizados y exponer una API HTTP local para Stream Deck.

🎯 Objetivo

Centralizar en una sola interfaz dinámica todo el flujo de operación de un stream competitivo profesional.

📥 Entradas

API keys, archivos locales JSON, datos consumidos de Challonge/start.gg, y comandos manuales/Stream Deck.

📤 Salidas

Overlays web elegantes, automatización sobre OBS, notificaciones a Twitch/Nightbot.

3. Módulos explicados uno por uno

3.1 main.js (backend de escritorio)

  • Levanta Electron y desactiva el menú superior por defecto para mantener aspecto app.
  • Orquesta la persistencia local leyendo/escribiendo en Documentos/js/scoreboard-config.json.
  • Canal IPC usando ipcMain.handle(...) para abrir comunicación total entre NodeJS y la vista UI.
  • Contiene las capas de red externas: WebSockets con OBS, APIs a Challonge/start.gg, e instancias TMI de Twitch.
  • Inicializa el servidor HTTP local en puerto 3001 diseñado para integrarse con Stream Deck o Touch Portal.

3.4 js/scoreboard.js (frontend principal)

  • Motor de estados principal del Dashboard Front-End.
  • Navegación fluida de pestañas, persistiendo inputs a JSON.
  • Delegación de interacciones OAuth para Nightbot y generación de comandos vía interfaz gráfica de usuario.

3.9 example/ (overlays)

  • El punto final del diseño visual para espectadores. HTML/JS/CSS listos para ser añadidos en fuentes "Browser Source" de OBS.
  • Estrategia Polling/Watcher: los overlays se hidratan periódicamente apuntando a archivos locales JSON modificados por la app principal.

7.1 Estado unificado: state.json

Todos los datos de la aplicación se consolidan en un único archivo Documentos/js/state.json organizado en secciones independientes. Cuando se guarda una sección (por ejemplo, scoreboard), el sistema lee el archivo completo, actualiza solo esa sección y lo escribe de vuelta. El resto de secciones queda intacto — actualizar los marcadores nunca borra los comentaristas ni el timer.

Estructura del archivo

{
  "scoreboard": {
    "player1": "...", "player2": "...",
    "score1": 0,      "score2": 0,
    "tag1": "...",    "tag2": "...",
    "char1": "...",   "char2": "...",
    "game": "...",    "round": "...",
    "country1": "",   "country2": ""
  },
  "casters": {
    "comentaristas": [
      { "nombre": "...", "twitter": "..." },
      { "nombre": "...", "twitter": "..." }
    ]
  },
  "timer": {
    "timerEndTimestamp": null
  },
  "top8": { ... },
  "bracket": { ... }
}

Patrón merge-on-write

Cada llamada a save-json sigue este flujo:

// Ejemplo: guardar scoreboard sin tocar casters ni timer
saveJsonData(nuevoScoreboard, 'scoreboard')
  1. Lee state.json  →  { scoreboard: {...}, casters: {...}, timer: {...}, ... }
  2. Actualiza solo  →  state['scoreboard'] = nuevoScoreboard
  3. Escribe back    →  state.json completo con casters y timer intactos
  4. Side-write      →  scoreboard.json individual (compatibilidad overlays OBS)

Secciones disponibles

SecciónGuarda / Carga conSide-write a
scoreboardsave-json / load-jsonDocumentos/js/scoreboard.json
casterssave-json / load-jsonDocumentos/js/casters.json
timersave-json / load-jsonDocumentos/js/timer.json

Migración automática

Si state.json no existe al arrancar la app, la función migrateToState() en main.js lo crea automáticamente leyendo los archivos individuales existentes, sin pérdida de datos.

Override de rutas

Es posible personalizar la ubicación de state.json añadiendo la clave state en el objeto rutas de scoreboard-config.json: 

// Documentos/js/scoreboard-config.json
{
  "rutas": {
    "state": "C:/ruta/personalizada/state.json"
  }
}

8. Tutorial rápido general

Requisitos Previos Básicos

  • OS: Windows 10/11.
  • Paquetería: NodeJS (npm incluido).
  • Hardware: OBS Studio v28+ (WebSocket v5.x).

Instalación y Arranque

Paso 1: Instalar librerías nativas requeridas 
npm install
Paso 2: Inicializar en entorno development (interactivo) 
npm start

Nota: Si usas .bat, corrobora la alineación de las rutas base del OS.

Conectar con OBS

  1. Abre OBS, despliega Herramientas > Configuración del servidor WebSocket. Asigna puerto y clave.
  2. En la App -> pestaña "OBS", provee dichas credenciales para abrir la conexión constante segura.
  3. En OBS, suma las fuentes de navegador Browser Source direccionando al filesystem.

8.1 Crear un Overlay HTML que lea datos de la app

Los overlays son archivos HTML que OBS carga como Browser Source. Su única responsabilidad es leer un JSON local y renderizar los datos. La app escribe ese JSON cada vez que el usuario guarda cambios.

Regla de oro: el overlay NUNCA escribe datos. Solo lee en polling (setInterval).

Opción A — Leer el archivo individual (side-write)

Para cada sección existe un archivo JSON independiente en Documentos/js/<sección>.json que la app actualiza automáticamente al guardar. Es la opción más sencilla: el overlay apunta a ese archivo y solo trabaja con los datos de su propia sección. 

// scoreboard-overlay.js — patrón base
const JSON_PATH = 'C:/Users/TU_USUARIO/Documents/js/scoreboard.json';
// Para casters: .../Documents/js/casters.json
// Para top8:    .../Documents/js/top8.json
// Para timer:   .../Documents/js/timer.json

let lastHash = '';

function updateOverlay() {
  fetch(JSON_PATH + '?t=' + Date.now())   // ?t= evita caché del navegador
    .then(r => r.json())
    .then(data => {
      const hash = JSON.stringify(data);
      if (hash === lastHash) return;       // sin cambios, no re-renderiza
      lastHash = hash;
      renderScoreboard(data);             // tu función que escribe en el DOM
    })
    .catch(() => {});                     // silencia errores de red en OBS
}

updateOverlay();
setInterval(updateOverlay, 2000);         // polling cada 2 segundos

Opción B — Leer state.json y extraer la sección

Si prefieres un solo punto de lectura para todos los overlays, apunta a state.json y accede a data.scoreboard, data.casters, etc. 

// overlay-genérico.js — lee state.json completo
const STATE_PATH = 'C:/Users/TU_USUARIO/Documents/js/state.json';
const SECCION    = 'scoreboard';  // cambia a 'casters', 'top8', 'timer', 'bracket'

let lastHash = '';

function updateOverlay() {
  fetch(STATE_PATH + '?t=' + Date.now())
    .then(r => r.json())
    .then(state => {
      const data = state[SECCION] || {};
      const hash = JSON.stringify(data);
      if (hash === lastHash) return;
      lastHash = hash;
      render(data);
    })
    .catch(() => {});
}

updateOverlay();
setInterval(updateOverlay, 2000);

Rutas correctas según el overlay

La ubicación física de los archivos depende del OS. La app siempre escribe en app.getPath('documents') + '/js/' a menos que se haya sobreescrito la ruta en scoreboard-config.json.

Sección Ruta por defecto (Windows) Variable en overlay
scoreboard C:/Users/<user>/Documents/js/scoreboard.json data.player1, data.score1 …
casters C:/Users/<user>/Documents/js/casters.json data.comentaristas[0].nombre …
top8 C:/Users/<user>/Documents/js/top8.json data.players[n] …
timer C:/Users/<user>/Documents/js/timer.json data.timerEndTimestamp …
bracket <raíz app>/example/json/bracket.json data.players, data.matches …
todo C:/Users/<user>/Documents/js/state.json state.scoreboard, state.casters …

⚠️ OBS y rutas locales

OBS bloquea fetch() a rutas file:/// por seguridad. Tienes dos alternativas para que el overlay lea los archivos correctamente:

  1. Activar "Acceso a archivos locales" en la configuración de la Browser Source de OBS (casilla "Allow access to local files"). Con esto fetch('C:/...') funciona sin cambios.
  2. Servir los archivos con el servidor HTTP de la app. La app expone http://localhost:3001; si se añade un endpoint GET /state o GET /state/:seccion, el overlay puede hacer fetch('http://localhost:3001/state/scoreboard') sin necesitar permisos especiales en OBS.

10. Troubleshooting / Típicos Problemas

App estática - NO actualizan Overlays

  • Posible colisión de permisos del SO. Revisar si la ruta de escritura configurada no tiene un candado o un Antivirus bloqueando fs.writeFile.
  • Error humano mapeo OBS: verificar la URL exacta del archivo html inyectado a la escena.

Stream Deck ignora los comandos

  • No levantar el backend npm start causará que localhost:3001 no exista.
  • Colisión de puertos. Quizás hay otra app consumiendo el tcp/3001.

11. Roadmap / Mejoras Recomendadas

  • Completar migración a contextIsolation: true blindando seguridad en IPC renderers.
  • Incorporar parsing validado por esquemas (Zod o AJV) al leer y escribir JSON locales para evitar corruptabilidad.
  • Generar test cases y Jest units para lógica utils.js en transformaciones de string.
  • Modificar el marcador desde la app (diseño visual del marcador en el HTML).
